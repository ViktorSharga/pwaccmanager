🔥 Critical Bug Fixes

Task 1: Settings Persistence in Standalone EXE

CRITICAL

Issue: When application is compiled as standalone .exe file, settings are not being saved/loaded properly.

Technical Details:

  • Settings likely not persisting due to file path issues in compiled environment
  • May be trying to write to protected directories or incorrect relative paths
  • Need to implement proper data directory detection for executable vs development environment

Requirements:

  • Detect if running from .exe or development environment
  • Use appropriate data storage location (AppData/Documents for .exe)
  • Implement fallback mechanism for settings storage
  • Test settings persistence across application restarts in compiled form
  • Ensure configuration files are created with proper permissions

Expected Outcome:

Settings save and load correctly in both development and compiled .exe versions

Task 2: Three Dots Menu Functionality

HIGH

Issue: Context menu (three dots menu) is not responding to user interactions.

Technical Details:

Expected Outcome:

Three dots menu fully functional with all menu items working correctly

Task 3: Process Launch Delay Management

HIGH

Issue: When launching multiple game windows simultaneously, process IDs are not being saved correctly due to race conditions.

Technical Details:

  • Concurrent launches causing process tracking conflicts
  • Need sequential launching with proper delays
  • Process ID assignment getting mixed up between instances

Requirements:

  • Implement queue-based launching system
  • Add configurable delay between consecutive launches
  • Ensure each process is fully started before launching next
  • Implement proper process ID tracking and assignment
  • Add launch status feedback to user
  • Handle launch failures gracefully without affecting other launches

Expected Outcome:

Multiple account launches work reliably with correct process tracking

Task 10: Fix Table Field Editing and Implement Clipboard Copy

HIGH

Issue: Table fields incorrectly enter edit mode without saving changes, and login/password fields need clipboard copy functionality instead of editing.

Problem 1: Disable Table Field Editing

Current Behavior:
  • Clicking on table fields triggers edit mode
  • Changes made in edit mode are not saved to data
  • Creates confusing user experience with non-functional editing
Technical Requirements:
  • Disable inline editing for all text fields in the table
  • Remove edit mode styling and cursor changes
  • Ensure table cells are read-only and non-interactive for text content
  • Maintain proper visual styling for non-editable cells
  • Keep existing selection functionality for rows (checkboxes should still work)
Implementation Details:
  • Remove or disable contentEditable attributes from table cells
  • Update CSS to remove text cursor (cursor: default instead of cursor: text)
  • Remove any event handlers that trigger edit mode
  • Ensure proper styling for read-only table cells

Problem 2: Clipboard Copy for Login/Password Fields

New Behavior Requirements:
  • Click on Login field → Copy login/username to clipboard
  • Click on Password field → Copy password to clipboard
  • Show visual feedback when copy operation succeeds
  • Handle copy operation failures gracefully
  • Add hover indicators to show fields are clickable for copying
Technical Implementation:
  • Implement clipboard copy functionality using navigator.clipboard.writeText()
  • Add fallback for older browsers using document.execCommand('copy')
  • Create click event handlers specifically for Login and Password columns
  • Add visual feedback (toast notification, temporary highlight, or tooltip)
  • Implement proper error handling for clipboard access failures
  • Add hover styling to indicate clickable fields
User Experience Enhancements:
  • Add subtle hover effects on Login/Password fields
  • Show temporary "Copied!" message or icon
  • Use appropriate cursor style (cursor: pointer) for copy-enabled fields
  • Consider adding small copy icon overlay on hover
  • Ensure accessibility with proper ARIA labels
Security Considerations:
  • Ensure clipboard copy works only for visible data
  • Handle password visibility settings appropriately
  • Add user permission handling for clipboard access
  • Consider adding confirmation for password copying (optional)

Expected Outcome:

  • Table provides clean, read-only display of account data
  • Login and password fields provide one-click copying to clipboard
  • Clear visual feedback for copy operations
  • Improved user workflow for credential management

  • Add import preview showing accounts to be imported
